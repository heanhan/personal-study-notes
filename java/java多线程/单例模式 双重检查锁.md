#### java的单例模式-双重检查锁

```java
//代码示例
public class Singleton {
  //使用volatile 关键字
  private static volatile Singleleton instance=null;
  //构造函数私有化
  private Singleton(){};
  //获取单例的实例对象
  public Singleton getInstance(){
    //第一次判断实例对象
    if(instace==null){
      //第二次判断实例对象，使用sychronized代码块进行加锁
      sychronized(Singleton.class){
        if(instance ==null){
          instance =new Singleton();
        }
      }
    }
    return instance;
  }
}
```

为什么是双重校验实现单例模式呢

```
第一次校验：这个是为了代码提高代码执行效率，由于单例模式只要第一次创建即可，所以当创建一个实例后，再次调用getInstance()；方法获取实例，就不需要再次进入同步代码块，不用锁竞争直接返回前面创建好的单例实例。
```
